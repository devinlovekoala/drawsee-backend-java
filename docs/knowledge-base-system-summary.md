# 知识库系统实现总结

## 系统架构

知识库系统的实现采用了分层架构，主要包括：

- 控制器层（Controller）：根据用户角色提供不同的API入口
- 服务层（Service）：实现核心业务逻辑和权限控制
- 数据访问层（Repository）：操作MongoDB存储知识库和知识点数据

## 用户角色与权限

系统支持三类用户角色，每类用户具有明确的权限边界：

### 管理员用户
- 可以创建和管理自己的知识库
- 可以查看系统中所有知识库（对他人创建的知识库只有只读权限）
- 在后台管理系统中有专用的知识库管理界面

### 教师用户
- 可以创建和管理自己的知识库
- 只能查看自己创建的或已加入的知识库
- 在后台管理系统中有专用的知识库管理界面

### 学生用户
- 通过邀请码加入知识库
- 只能查看已加入的知识库（只读权限）
- 在后台管理系统中可以访问已加入的知识库内容

## 邀请码机制

- 每个知识库都有一个唯一的8位邀请码
- 用户可以通过邀请码加入知识库
- 新用户注册时可以使用邀请码，注册成功后自动加入对应知识库

## 关键API接口

系统按照不同用户角色提供了专用的API：

1. **管理员接口**：`/admin/knowledge/base`
   - 创建知识库
   - 查看所有知识库列表
   - 管理自己创建的知识库

2. **教师接口**：`/teacher/knowledge/base`
   - 创建知识库
   - 管理自己的知识库
   - 上传教学资源

3. **学生接口**：`/student/knowledge/base`
   - 查看已加入的知识库
   - 浏览知识点和资源

4. **通用接口**：`/knowledge/base`
   - 根据用户角色自动重定向到相应操作
   - 提供共享资源访问

## 权限控制实现

1. **知识库访问权限**
   - 公开知识库：所有用户可访问
   - 私有知识库：仅创建者和成员可访问

2. **知识库编辑权限**
   - 仅知识库创建者可以编辑和删除知识库内容
   - 管理员只能编辑自己创建的知识库，不能编辑其他管理员或教师创建的知识库

3. **知识点管理权限**
   - 仅知识库创建者可以添加、编辑、删除知识点
   - 所有有权访问知识库的用户都可以查看知识点

## 系统特色

1. **严格的权限隔离**：确保用户只能操作自己有权限的内容
2. **灵活的邀请码机制**：便于知识库内容的分享和管理
3. **知识图谱展示**：直观呈现知识点之间的关系
4. **多类型资源支持**：支持文档、视频和B站资源等多种格式
