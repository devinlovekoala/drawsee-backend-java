# M1 交付物与预案（基线不变）

- 目标：在不影响现有业务的前提下，完成“只读/告警级”的依赖安全扫描与日志微调能力，产出风险清单、回滚预案与最小可运行链路自测清单。

## 一、基线不变声明
- 不改对外接口契约与运行参数。
- 不调整默认 `logging` 行为；新增 `observe` Profile 仅在显式启用时生效。
- 安全加固（密钥外置、CORS、Actuator 收敛）暂缓，记录为后续事项。

## 二、风险清单（记录）
- 配置中存在敏感信息（数据库、Redis、RabbitMQ、Minio、模型 API Key）。本阶段不改动，后续建议：环境变量/密钥管理器、分环境覆盖、最小权限与审计。
- 依赖组件 CVE 风险：通过 `security-scan` Profile 执行依赖扫描，仅输出报告不阻断构建。
- 日志噪声与磁盘占用：默认不变；`observe` 仅用于临时问题定位。
- 构建稳定性：扫描下载 CVE 数据可能较慢，建议在离线缓存或 CI 定时任务内执行。

## 三、回滚预案
- 所有改动为增量且默认不生效：
  - 移除或不启用 `-Psecurity-scan` 即可不执行扫描。
  - 不添加 `observe` Profile 即保持日志为原始行为。
- 若扫描插件引起构建耗时过长，可临时在命令行追加 `-Ddependency-check.skip=true`。

## 四、最小可运行链路自测清单
- 启动：`mvn spring-boot:run`（或 `mvn -DskipTests package && java -jar target/*.jar`）。
- 基础链路：
  - Redis/RabbitMQ/Minio/MySQL 可达性检查（连接失败仅记录报错）。
  - AI 任务最小闭环（创建→消费→流式输出→完成）。
- 日志：默认级别保持不变；如需排查，启用 `observe` Profile（见下）。

## 五、使用指南
### 1. 依赖安全扫描（只读/告警级）
- 命令：
  - 依赖扫描与静态分析（建议本地或 CI 手动触发）：
    - `mvn -Psecurity-scan -DskipTests verify`
  - 仅查看依赖可升级情况：
    - `mvn versions:display-dependency-updates`
    - `mvn versions:display-plugin-updates`
- 输出：
  - 依赖漏洞报告：`target/dependency-check-report.html` 与 `target/dependency-check-report.json`
  - SpotBugs 报告：`target/spotbugsXml.html`（或 `spotbugsXml.xml`）
- 不阻断构建：已将 `failOnCVSS` 设为超高阈值，并关闭 SpotBugs failOnError。

### 2. 观察日志 Profile（按需启用）
- 启用方式（叠加现有 `prod/dev/test`）：
  - `--spring.profiles.active=prod,observe`
- 内容：提高 `cn.yifan.drawsee`、`org.springframework.web`、`org.springframework.amqp`、`org.redisson`、`io.minio`、`org.mybatis` 的日志级别至 `info`，仅用于问题定位。

## 六、后续事项（记录）
- 安全加固：密钥外置、精简 Actuator 暴露、CORS 收敛、日志脱敏。
- 依赖升级：在 M2/M3 推进前，优先评估关键依赖的兼容性与性能影响。
