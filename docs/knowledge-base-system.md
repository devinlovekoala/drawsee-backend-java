# 知识库系统设计文档

## 系统概述

知识库系统是DrawSee平台的核心功能之一，允许管理员、教师和学生用户创建、管理和访问知识库资源。系统支持知识点的组织、资源的上传和管理，以及知识图谱的可视化展示。

## 用户角色和权限

系统支持三种用户角色，每种角色具有不同的权限：

### 1. 管理员 (Admin)

- 创建自己的知识库
- 编辑和删除自己创建的知识库（不能编辑其他人创建的知识库）
- 查看平台上的所有知识库（对他人创建的知识库只读不可编辑）
- 设置知识库的发布状态

### 2. 教师 (Teacher)

- 创建自己的知识库
- 编辑和管理自己创建的知识库
- 添加、编辑和删除知识点
- 上传和管理教学资源
- 查看自己创建的和加入的知识库

### 3. 学生 (Student)

- 通过邀请码加入知识库
- 查看已加入的知识库内容（只读权限）
- 浏览知识点和资源
- 访问知识图谱
- 注册时通过邀请码自动加入知识库

## 功能列表

### 知识库管理

1. **创建知识库**
   - 管理员和教师可以创建知识库
   - 设置知识库名称、描述和学科分类
   - 系统自动生成唯一的8位邀请码

2. **加入知识库**
   - 用户通过邀请码加入知识库
   - 学生注册时可以使用邀请码自动加入知识库

3. **查看知识库**
   - 管理员：查看所有知识库，但只能编辑自己创建的
   - 教师：查看自己创建的和加入的知识库
   - 学生：查看自己通过邀请码加入的知识库

### 知识点管理

1. **添加知识点**
   - 在知识库中创建新的知识点（仅知识库创建者可操作）
   - 设置知识点名称、学科、级别和父子关系

2. **编辑知识点**
   - 更新知识点的信息和关联关系（仅知识库创建者可操作）
   - 管理知识点的资源

3. **删除知识点**
   - 删除知识点及其关联资源（仅知识库创建者可操作）

### 资源管理

1. **上传资源**
   - 支持文档、图片、视频等多种格式的教学资源
   - 每个知识点可以关联多个资源

2. **添加B站资源**
   - 直接关联B站视频作为教学资源

3. **资源统计**
   - 查看知识点的资源统计信息

### 知识图谱

1. **查看知识图谱**
   - 图形化展示知识点之间的关系
   - 直观了解知识体系结构

## API接口设计

系统按照不同角色提供了专用的API接口：

### 管理员接口

- 前缀：`/admin/knowledge/base`
- 主要功能：
  - 创建和管理自己的知识库
  - 查看所有知识库（只读权限）
  - 设置知识库发布状态

### 教师接口

- 前缀：`/teacher/knowledge/base`
- 主要功能：
  - 创建和管理自己的知识库
  - 管理知识点和资源

### 学生接口

- 前缀：`/student/knowledge/base`
- 主要功能：
  - 加入知识库
  - 查看已加入的知识库和资源（只读）

### 通用接口

- 前缀：`/knowledge/base`
- 主要功能：
  - 根据用户角色提供相应的功能
  - 提供通用的资源访问接口

## 邀请码机制

1. **生成规则**
   - 8位字符，排除易混淆字符
   - 自动检查唯一性
   - 每个知识库有唯一的邀请码

2. **使用方式**
   - 直接加入知识库：已有账户的用户通过邀请码加入知识库
   - 注册时加入：新用户注册时使用邀请码，成功注册后自动加入知识库

## 使用流程

### 管理员

1. 登录系统，进入管理界面
2. 创建自己的知识库，设置基本信息
3. 添加知识点，构建知识体系
4. 上传资源，丰富知识内容
5. 查看系统中的所有知识库（对他人创建的知识库只读）

### 教师

1. 登录系统，进入教师界面
2. 创建自己的知识库
3. 添加和组织知识点
4. 上传教学资源
5. 将邀请码分享给学生

### 学生

1. 通过教师提供的邀请码注册或登录系统
2. 自动加入或手动输入邀请码加入相应知识库
3. 浏览知识库内容和知识图谱
4. 学习相关知识和资源

## 总结

知识库系统提供了一个友好的环境，让管理员、教师和学生能够有效地组织、分享和获取知识。通过精心设计的权限管理和邀请码机制，系统确保了资源的安全性和可控性，同时保证了用户操作的友好性。

每类用户都有明确定义的权限边界：
- 管理员可以查看所有知识库，但只能编辑自己创建的知识库
- 教师可以创建和管理自己的知识库
- 学生可以通过邀请码加入并查看知识库内容

所有知识库都有唯一的8位邀请码，确保只有获得邀请码的用户才能访问相应的知识库内容。
