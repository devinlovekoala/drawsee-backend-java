## 集成概览

### 系统架构对比

**现有电路分析系统**:
```
用户上传电路
  ↓
CircuitDesign (Java对象)
  ↓
SpiceConverter → SPICE Netlist
  ↓
CircuitAnalysisWorkFlow + AI分析
  ↓
流式响应 + 追问预判
```

**Agentic RAG v2 系统**:
```
用户查询 + (可选)图片/Netlist
  ↓
AdaptiveChannelRouter
  ├─ INPUT_TYPE分类
  └─ INTENT分类
  ↓
频道路由 (KNOWLEDGE/FORMULA/NETLIST/VISION)
  ↓
专业处理 + 结构化输出
  ↓
(可选)再路由到其他频道
```

### 集成策略

**设计原则**:
- **互补而非替代**: Agentic RAG v2作为现有系统的增强层
- **数据复用**: 共享CircuitDesign和SPICE格式
- **统一入口**: 通过AgenticRagController提供统一API
- **逐步迁移**: 先集成，后优化，最终融合

### 1. 数据格式对齐

**优势**:
- ✅ 100%兼容现有SPICE格式

```java
/**
 * 验证SPICE Netlist并提取结构信息
 */
public Map<String, Object> validateNetlist(
    String netlist,
    Long userId,
    String classId
) {
    try {
        // 构建请求
        Map<String, Object> request = new HashMap<>();
        request.put("query", netlist);
        request.put("knowledgeBaseIds", Collections.emptyList());
        request.put("context", Map.of("validation_only", true));

        // 调用Python Agentic RAG API
        String pythonApiUrl = pythonAgenticRagUrl + "/netlist/validate";

        // 发送POST请求
        String jsonRequest = objectMapper.writeValueAsString(request);
        HttpURLConnection connection = createConnection(
            pythonApiUrl,
            "POST",
            userId,
            classId
        );

        try (OutputStream os = connection.getOutputStream()) {
            os.write(jsonRequest.getBytes(StandardCharsets.UTF_8));
        }

        // 解析响应
        String response = readResponse(connection);
        return objectMapper.readValue(response, Map.class);

    } catch (Exception e) {
        logger.error("Netlist验证失败", e);
        return Map.of("success", false, "error", e.getMessage());
    }
}
```

```python
import hashlib
from functools import lru_cache

@lru_cache(maxsize=100)
def _get_simulation_cache_key(netlist: str) -> str:
    return hashlib.md5(netlist.encode()).hexdigest()

async def _run_simulation(self, netlist: str, parse_result: Dict):
    cache_key = self._get_simulation_cache_key(netlist)

    # 检查Redis缓存
    cached = await redis_client.get(f"sim:{cache_key}")
    if cached:
        return json.loads(cached)

    # 运行仿真
    result = await self._run_simulation_impl(netlist, parse_result)

    # 缓存结果(1小时)
    await redis_client.setex(
        f"sim:{cache_key}",
        3600,
        json.dumps(result)
    )

    return result
```

## VisionChannel与现有电路识别系统的集成

### 1. 现有电路识别系统架构

**核心组件**: `CircuitImageNetlistParser.java`

**识别流程**:
```
1. 用户上传电路图片 (ToolController.java)
   POST /tool/recognize_circuit/upload

2. 存储图片URL到OSS/S3

3. 调用AI视觉模型识别电路元件和连接

4. 返回特定格式的文本描述:
   TITLE AI识别电路
   COMP id="R1" type="resistor" x="100" y="50"
   WIRE id="W1" source="R1.port1" target="C1.port1"

5. CircuitImageNetlistParser解析为CircuitDesign对象

6. 保存到数据库 (CircuitDesignMapper)

7. 可选: 生成SPICE netlist进行分析
```

**支持的元件**: resistor, capacitor, inductor, dc_source, ac_source, diode, transistor_npn, transistor_pnp, opamp, ground

### 2. VisionChannel的设计优势

**与现有系统的差异**:

| 特性 | 现有系统 | VisionChannel |
|-----|---------|--------------|
| **输出格式** | 固定的COMP/WIRE文本 | 灵活的JSON结构 |
| **识别范围** | 仅元件和连接 | 元件+拓扑+参数+意图 |
| **后续处理** | 需手动选择分析 | 智能再路由到其他频道 |
| **用户体验** | 多步操作 | 一次上传，自动处理 |
| **可扩展性** | 依赖特定格式 | 支持多种视觉模型 |

**VisionChannel的创新点**:

1. **智能再路由**:
```python

{
    "recognition_result": "这是一个共发射极放大电路...",

    "reroute_reason": "用户询问工作原理，需要深度分析"
}
```

2. **多模态理解**:
- 不仅识别"是什么"，还理解"用户想做什么"
- 根据用户问题决定下一步处理
- 避免用户重复上传图片

```java
@PostMapping("/recognize_circuit")
public ResponseEntity<Map<String, Object>> recognizeCircuit(
    @RequestParam String imageUrl,
    @RequestParam(required = false, defaultValue = "识别这个电路") String query
) {
    try {
        // 步骤1: 调用VisionChannel进行识别
        Map<String, Object> visionResult = agenticRagService.recognizeCircuitWithVision(
            imageUrl,
            query,
            StpUtil.getLoginIdAsLong()
        );

        // 步骤2: 提取识别结果
        Map<String, Object> recognition =
            (Map<String, Object>) visionResult.get("recognition_result");

        String rawRecognition = (String) recognition.get("raw_recognition");
        List<String> components = (List<String>) recognition.get("components");
        String topology = (String) recognition.get("topology");

        // 步骤3: 转换为CircuitImageNetlistParser期望的格式
        String formattedNetlist = convertVisionToNetlistFormat(
            components,
            topology
        );

        // 步骤4: 使用现有解析器
        CircuitDesign design = circuitImageNetlistParser.parse(formattedNetlist);

        // 步骤5: 保存到数据库
        Long circuitId = circuitDesignService.saveCircuitDesign(design, userId);

        // 步骤6: 返回结果
        return ResponseEntity.ok(Map.of(
            "success", true,
            "circuitId", circuitId,
            "design", design,
            "visionInsight", topology  // 额外的视觉理解
        ));

    } catch (Exception e) {
        logger.error("电路识别失败", e);
        return ResponseEntity.status(500).body(
            Map.of("success", false, "error", e.getMessage())
        );
    }
}

private String convertVisionToNetlistFormat(
    List<String> components,
    String topology
) {
    StringBuilder sb = new StringBuilder();
    sb.append("TITLE AI识别电路\n");
    sb.append("DESCRIPTION ").append(topology).append("\n");

    // 将VisionChannel的components列表转为COMP格式
    for (int i = 0; i < components.size(); i++) {
        String comp = components.get(i);
        // 假设格式: "R1 (电阻, 1kΩ)"
        // 转换为: COMP id="R1" type="resistor" label="1k"
        sb.append(parseComponentToNetlist(comp, i));
    }

    return sb.toString();
}
```

### 4. 数据格式映射

**VisionChannel输出 → CircuitDesign映射**:

```python
# vision_channel.py
def _format_for_legacy_system(self, recognition: Dict) -> str:
    """
    将VisionChannel的识别结果转换为CircuitImageNetlistParser格式
    """
    lines = []
    lines.append(f"TITLE {recognition.get('circuit_type', 'AI识别电路')}")
    lines.append(f"DESCRIPTION {recognition.get('topology', '')}")

    # 元件转换
    for i, comp in enumerate(recognition.get('components', [])):
        comp_id = comp.get('name', f'U{i+1}')
        comp_type = self._map_component_type(comp.get('type'))

        # 估算位置(基于识别顺序)
        x = 100 + (i % 5) * 150
        y = 100 + (i // 5) * 150

        lines.append(
            f'COMP id="{comp_id}" type="{comp_type}" '
            f'x="{x}" y="{y}" label="{comp.get("value", "")}"'
        )

    # 连接转换(基于nodes信息)
    wire_id = 1
    for comp in recognition.get('components', []):
        nodes = comp.get('nodes', [])
        if len(nodes) >= 2:
            lines.append(
                f'WIRE id="W{wire_id}" '
                f'source="{comp["name"]}.port1" '
                f'target="N{nodes[1]}.node"'
            )
            wire_id += 1

    return '\n'.join(lines)

def _map_component_type(self, vision_type: str) -> str:
    """映射元件类型名称"""
    mapping = {
        "R": "resistor",
        "C": "capacitor",
        "L": "inductor",
        "V": "dc_source",
        "I": "current_source",
        "D": "diode",
        "Q": "transistor_npn",
        "M": "mosfet"
    }
    return mapping.get(vision_type, "unknown")
```
