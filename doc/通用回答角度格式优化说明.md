# 通用回答角度格式优化说明

## 背景

在Drawsee平台的通用对话模式中，我们原先使用JSON格式返回回答角度数据，而在知识问答模式中使用了更加清晰的文本格式返回知识点。为了统一系统设计和提高代码可维护性，我们对通用对话模式中的回答角度数据格式进行了优化。

## 优化内容

### 1. 提示词模板优化

**修改文件**: `/src/main/resources/prompt/answer-point.txt`

将原有的JSON输出格式修改为直接的文本格式输出，使其与知识问答模式的格式保持一致：

**原格式（JSON）**:
```json
[
  {
    "title": "角度标题1",
    "description": "对这个角度的简短一句话描述"
  },
  {
    "title": "角度标题2",
    "description": "对这个角度的简短一句话描述"
  }
]
```

**新格式（文本）**:
```
角度1：角度标题1
对这个角度的简短一句话描述

角度2：角度标题2
对这个角度的简短一句话描述
```

### 2. 解析逻辑优化

**修改文件**: `/src/main/java/cn/yifan/drawsee/worker/WorkFlow.java`

对`createAnswerPointNodes`方法进行了优化，使其能够识别和解析新的文本格式，同时保留对旧有JSON格式的兼容性：

1. **双重解析策略**:
   - 首先尝试以JSON格式解析
   - 如果JSON解析失败，则使用文本格式解析

2. **文本格式解析逻辑**:
   - 按行分割文本
   - 识别"角度X：[标题]"格式的行作为标题
   - 标题后的行作为描述
   - 空行作为分隔符

3. **封装重复代码**:
   - 创建`createSingleAnswerPointNode`方法处理单个角度节点的创建

## 优点

1. **统一数据格式**: 使通用对话模式与知识问答模式的数据格式保持一致
2. **降低复杂度**: 文本格式更易于阅读和调试
3. **兼容性保障**: 保留对旧有JSON格式的兼容支持，确保平滑过渡
4. **提高可维护性**: 统一处理流程，简化代码结构

## 实现要点

1. **正则表达式匹配**: 使用`line.matches("^角度\\d+：.+")`识别角度标题行
2. **状态追踪**: 使用`currentTitle`和`currentDescription`跟踪当前处理的角度信息
3. **边界处理**: 特别处理文本结尾的最后一个角度
4. **错误处理**: 完善的异常捕获和日志记录，确保解析失败不会影响整体流程

## 后续工作

1. **前端显示优化**: 确认前端能正确识别和展示新格式的角度数据
2. **性能监控**: 关注解析性能，确保新的解析逻辑不会带来明显的性能开销
3. **用户反馈收集**: 收集用户对新格式的反馈，进一步优化用户体验
