# 参数错误(parameter error)问题分析

## 问题描述
在发送GENERAL_DETAIL类型的AI任务时，前端组件接收到"parameter error"错误信息。

## 问题原因分析

### 1. CreateAiTaskDTO参数验证

查看`CreateAiTaskDTO`类，发现其中的`prompt`字段有`@NotBlank`注解：

```java
@NotBlank(message = "提示词不能为空")
private String prompt;
```

这表明系统要求这个字段不能为空白，而我们的业务逻辑更新后不再需要在GENERAL_DETAIL任务中从前端传入prompt，这导致了验证失败。

### 2. 参数验证逻辑

在`FlowService.createTask`方法中有以下验证逻辑：

```java
// 参数校验
if (createAiTaskDTO.getConvId() != null && createAiTaskDTO.getParentId() == null) {
    throw new ApiException(ApiError.PARAM_ERROR);
}
```

这个校验要求，如果设置了`convId`，则必须设置`parentId`。

### 3. 异常处理

系统使用`GlobalExceptionHandler`进行统一异常处理，会将参数验证异常转换为"参数错误"(parameter error)的响应：

```java
// 参数校验异常
@ExceptionHandler(MethodArgumentNotValidException.class) // 方法上
public ResponseEntity<Result<Object>> handleMethodArgumentNotValidException(MethodArgumentNotValidException e) {
    return getErrorResponse(ApiError.PARAM_ERROR);
}
```

## 解决方案

### 短期解决方案

前端在发送GENERAL_DETAIL类型任务时，需确保：

1. `prompt`字段不为空（可以传入空字符串，但不能为null）
2. 确保正确设置了`parentId`（回答角度节点ID）
3. 确保设置了`convId`（会话ID）

### 长期解决方案

1. 修改`CreateAiTaskDTO`类，使`prompt`字段的验证条件根据任务类型动态变化，对于GENERAL_DETAIL等从父节点获取数据的任务类型允许为空
2. 更新API文档，明确说明不同任务类型所需的参数要求
3. 优化错误处理，提供更具体的错误消息，而不是通用的"参数错误"

## 前端代码示例

```typescript
// 创建详细回答任务
const createDetailTask = async (parentNodeId: string) => {
  setLoading(true);
  try {
    const response = await api.post('/flow/tasks', {
      type: 'GENERAL_DETAIL',
      prompt: "", // 提供一个空字符串而不是null
      promptParams: {}, // 空对象或不传
      convId: conversationId,
      parentId: parentNodeId
    });
    
    // 处理成功响应
    console.log('详细回答任务创建成功:', response.data);
    
  } catch (error) {
    console.error('创建详细回答任务失败:', error);
    // 提供更友好的错误提示
    message.error(error.response?.data?.message || '任务创建失败');
  } finally {
    setLoading(false);
  }
};
```
