我会给你一份教材目录页的文本内容，请提取其中的章节结构，并按照教材的组织方式构建知识点体系。

# 分析任务说明
目录页通常包含章节编号、标题和页码，你需要准确识别章节层级关系，并构建合理的知识结构。

# 分析规则
1. 精确识别章节编号和标题，例如："第一章 xxx"、"1.1 xxx"、"§1.2 xxx"等各种形式
2. 正确判断章节层级关系，区分章、节、小节等不同层级
3. 忽略页码信息，专注于章节结构本身
4. 根据标题内容推断知识点的核心内容和关键词
5. 保持原有的章节逻辑和层级关系
6. 对不同类型教材的章节特点进行适应性处理（如语文、数学、英语等学科特点）

# 适应不同学科和年级特点
* 小学教材：章节结构简单明了，标题通常较短
* 初中教材：章节层级开始增加，通常有2-3层结构
* 高中教材：章节结构更加复杂，可能有4-5层结构
* 语文教材：可能按单元、课文组织，标题常为文章名
* 数学教材：通常按章节、定理、公式等组织，编号规范
* 英语教材：可能按Unit、Module、Part等方式组织
* 理科教材：可能包含实验、探究等特殊章节
* 需要处理插图、附录、习题等特殊节点

# 输出格式要求
请输出JSON格式的知识点层级结构，每个节点包含以下字段：
- id: 唯一标识（使用UUID）
- title: 章节标题，保留原始格式（包含编号）
- description: 根据标题推断的简要描述（若无法推断则可简单概括为"本节介绍[标题内容]"）
- level: 层级深度（1为章，2为节，3为小节，依此类推）
- parentId: 父节点ID（顶级节点为null）
- keywords: 从标题中提取的3-5个关键词
- children: 子章节列表（包含完整子节点结构）

# 特殊处理要求
1. 正确处理各种章节编号格式：数字（1、1.1）、汉字（一、第一章）、特殊符号（§1.2）等
2. 识别并去除页码信息（如"...15"、"第一章 绪论 1"中的数字）
3. 适应不同教材的组织逻辑（单元制、章节制等）
4. 对目录中可能出现的附录、习题集、参考文献等特殊部分也要正确分类
5. 处理可能的排版问题（如换行导致的标题断裂）
